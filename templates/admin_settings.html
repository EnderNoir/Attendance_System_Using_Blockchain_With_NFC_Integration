Email Notifications
Send attendance receipts to students and session summaries to teachers via Gmail SMTP.
Recommended: SendGrid Setup (Most Reliable for Live Sites)
Live Ready
1
Create a SendGrid Account
Sign up at sendgrid.com. The Free plan allows 100 emails/day, which is perfect for this system.
2
Verify a Sender Identity
Go to Settings → Sender Authentication. Complete "Single Sender Verification" by clicking the link in the email they send you.
⚠ Emails will fail until your sender email is verified by SendGrid
3
Generate API Key
Go to Settings → API Keys → Create API Key. Give it "Full Access" or "Mail Send" permissions. Copy the key (starts with SG.).
4
Enter Credentials Below
SMTP Host smtp.sendgrid.net
SMTP Port 587
SMTP Username apikey (literally use this word)
Password Your SendGrid API Key (SG.xxxx)
Why SendGrid? Gmail often blocks connections from cloud servers like Railway to prevent spam. SendGrid is designed for apps and is much more reliable for live deployments.
Enable Email Notifications
When enabled, students receive tap receipts and teachers receive session summaries.
Use 587 for SendGrid
For SendGrid, this must be exactly apikey
Your SendGrid API Key
Optional — defaults to Gmail address if blank
What Gets Sent
Overview of email notifications sent by DAVS.
Student Receipt
  • Sent on every card tap
  • Includes: name, subject, section, instructor
  • Attendance status (Present / Late / Absent / Excused)
  • Date and time of tap
  • Blockchain TX hash + Block # as tamper-proof proof
Teacher Summary
  • Sent when session ends
  • Session details: subject, section, time slot
  • Present / Late / Absent / Excused counts
  • Full student list with individual status
  • Blockchain TX hashes for all records
 Student receipt emails are only sent if the student has an email address registered in their profile. Teacher summary emails require the teacher's account to have an email address set.
