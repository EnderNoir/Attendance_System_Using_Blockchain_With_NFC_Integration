How DAVS Uses Blockchain for Attendance Records

This page uses real chain data from this project. In DAVS, attendance events are validated first, then written to the smart contract, and finally connected to a transaction hash and block for audit proof. You can explore each concept below in beginner-friendly language, with examples tied to your attendance workflow.

Why is this important for attendance?

If a record says a student was Present at 08:03, the corresponding blockchain transaction hash is fixed. If someone later changes that record to Absent without proper blockchain entry, the hash linkage will not match. That mismatch is a red flag during audits.

Block = container: each block contains transaction references, a nonce, its own hash, and the previous block hash. In this project, a block may include attendance transaction hashes from one or more class sessions.

Blockchain = linked blocks: each block points to the previous one. If a block changes, chain linkage is broken. Click any block card to inspect it. This is the same principle used by DAVS for trustworthy attendance evidence.
How this helps teachers and admins

During disputes (for example, a student claims they tapped on time), admins can cross-check the stored transaction hash and block metadata. If hashes and chain links are valid, the attendance trail is consistent.

Distributed concept: peers keep copies of ledger data. If one peer is behind, it syncs from the latest valid chain. This means attendance verification does not depend on only one viewer screen or one report export.
Token note for this project: attendance does not require students to transfer tokens. Blockchain transactions are used here as immutable record anchors. Students use NFC tap flow, while blockchain provides verifiable proof in the background.

Attendance Transaction Density

Simple glossary

Hash: digital fingerprint.

Nonce: changing number used to find valid hash patterns in mining demos.

Block: container of transaction references + metadata.

Chain: ordered, linked blocks.

How Blockchain Is Used In This Attendance System

1. NFC Tap

Student taps NFC card/phone. Example: 2022-000123 taps at 08:03 for Computer Programming 1.

2. Session Validation

DAVS checks active session, schedule window, and enrollment before accepting Present/Late status.

3. Blockchain Write

System writes attendance event to smart contract and receives tx hash + block number as audit reference.

4. Audit Verification

Reports can show tx hash and block details so admins can verify that records are consistent and untampered.

Why this is beginner-friendly in practice

Think of blockchain here like a permanent receipt system: when attendance is recorded, DAVS gets a receipt number (transaction hash) stored inside a sealed page (block). Sealed pages are linked in order, so changing an old page is easy to detect.
